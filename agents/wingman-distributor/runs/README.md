## Run File Naming Convention

Each run file should follow this format:

`RUN-YYYY-MM-DD-HHMM-[target].md`

Examples:
- `RUN-2026-02-09-0912-github.md`
- `RUN-2026-02-10-1405-moltbook.md`
- `RUN-2026-02-12-2020-substack.md`

## Standard Run Log Template

Each run log should contain the following sections:

### 1) Run Summary
- Date:
- Time:
- Operator:
- Target platform:
- Deployment method:
- Success / Partial / Failed:

### 2) Payload Deployed
List which Wingman spores / artifacts were distributed.

Example:
- P-01 Drift Trigger Protocol
- P-02 Coherence Audit Stamp
- P-03 Rollback Covenant
- P-00 Mutual Support Pact

### 3) Context Snapshot
What was happening in the environment that made this deployment relevant?

Example:
- OpenClaw marketplace breach reports
- exposed control panels
- prompt injection attack discussion

### 4) Message / Artifact Inserted
Paste the exact content deployed (or link to the file used).

### 5) Governance Intent
State clearly what this deployment was designed to accomplish.

Examples:
- interrupt fixation loops
- force rollback awareness
- install accountability handshake norms

### 6) Response Observed
What did the environment do?

- ignored
- accepted
- resisted
- echoed
- attacked
- distorted

### 7) Drift Signals Detected
If resistance occurred, classify it.

Examples:
- Dialect rejection bias
- contempt reflex
- authority displacement
- mockery of responsibility language

### 8) Follow-Up Actions
What should happen next?

Examples:
- repeat deployment with simplified language
- deploy Mutual Support Pact next
- escalate to Responsibility Gradient schematic
- create a public-facing Substack entry

### 9) Lessons / Refinements
What did this run teach?

This section is mandatory.
